# Lec2: 关系数据库

## 概述

关系数据库系统是目前使用最广泛的数据库系统。关系模型是关系数据库系统的基础，也是本课程的重点。

关系数据库系统与其他非关系数据库系统的区别：关系数据库系统**只有"关系"（或"表"、"二维表"）这一种数据结构**。

关系模型由三个部分组成：

- **数据结构**：关系（二维表）
- **数据操作**：关系操作（选择、投影、连接等）
- **完整性约束**：实体完整性、参照完整性、用户定义的完整性

## 2.1 关系数据结构及形式化定义

关系模型仅有单一的数据结构——**关系**。现实世界中的实体以及实体间的各种联系均用关系来表示。关系模型的逻辑结构是**二维表**，建立在集合代数的基础上。

### 2.1.1 域（Domain）

**定义**：域是一组具有相同数据类型的值的集合。

例如：

- 整数的集合
- 实数的集合
- {男, 女}
- {0, 1, 2, ..., 100}
- 长度不超过 20 的字符串的集合

### 2.1.2 笛卡尔积（Cartesian Product）

**定义**：给定一组域 D₁, D₂, ..., Dₙ（允许其中某些域相同），D₁, D₂, ..., Dₙ 的笛卡尔积定义为：

> D₁ × D₂ × ... × Dₙ = {(d₁, d₂, ..., dₙ) | dᵢ ∈ Dᵢ, i = 1, 2, ..., n}

例子：
设有两个集合（域）：
学生域 (D_1={张三, 李四})
课程域 (D_2={数据库, 操作系统, 计算机网络})
那么它们的笛卡尔积是：
$D_1 \times D_2 = {(张三, 数据库),(张三, 操作系统),(张三, 计算机网络),(李四, 数据库),(李四, 操作系统),(李四, 计算机网络)}$
其中：

- 每一个元素 (d₁, d₂, ..., dₙ) 叫做一个 **n 元组**（n-tuple），简称**元组**（Tuple）
- 元素中的每一个值 dᵢ 叫做一个**分量**（Component）
- 若 Dᵢ（i = 1, 2, ..., n）为有限集，其基数（Cardinal Number）为 mᵢ，则笛卡尔积的基数为 m₁ × m₂ × ... × mₙ(即所有可能的元组的数目)
- 笛卡尔积可表示为一张二维表

### 2.1.3 关系（Relation）

**定义**：D₁ × D₂ × ... × Dₙ 的子集叫做在域 D₁, D₂, ..., Dₙ 上的**关系**，表示为：

> R(D₁, D₂, ..., Dₙ)

其中：

- R 表示关系的名字
- n 是关系的**度**（Degree）或**目**
  - n = 1 时，称为单元关系（Unary Relation）
  - n = 2 时，称为二元关系（Binary Relation）
  - n > 2 时，称为多元关系（N-ary Relation）
- 关系中的每个元素是关系中的一个**元组**，通常用 t 表示
- 关系中元组的数目称为关系的**基数**

关系是一张**二维表**：

- 表的每行对应一个元组
- 在一个关系中，不同列可以对应**相同的域**，表的每列对应一个域，列称为**属性**（Attribute）
- 关系中的每一个属性都有一个名字，称为**属性名**。在同一个关系中，属性名互不相同。
- 在关系对应的二维表中，表中的第一行被称为是二维表的**表头**，里面填写的是各个属性的**属性名**。除第一行表头外，其他的每一行都是关系中的**一个元组**。
- n目关系必有n个属性
  ![1775648049282](image/lec2关系数据库/1775648049282.png)
  每一列可以是同一个域，但是必须要有不同的属性名来区分不同的列，例如：学生关系中的"姓名"和教师关系中的"姓名"虽然都是同一个域（字符串），但它们是不同的属性，必须有不同的属性名来区分。

若t是关系R中的一个元组，用t[A]表示t在属性A上的分量值（即t在属性A上的取值），并且t[A]的值必须在属性A的域$D_A$中。例如：学生关系中的一个元组t = (2019001, 张三, 男, 计算机科学与技术)，则t[姓名] = 张三。

### 2.1.4 关系的性质

关系的分类：

- 基本关系（基本表 或 基表）：**实际存在的表**，是实际存储数据的逻辑表示。
- 查询表：查询结果对应的表，结果数据不需要持久存储。
- 视图表：由基本表或其他视图表导出的表,是**虚表**,结果数据也不需要持久存储。
  表是SQL中的概念，关系是关系模型中的概念，关系模型中没有表的概念，但关系可以用表来表示。
  关系模型是一种逻辑数据模型。

关系是笛卡尔积的子集，而定义关系的域可以是无限的，因此关系可以是无限的。
但在我们使用关系作为关系数据模型的基本数据结构时，我们通常加入以下限制：必须是一个有限子集，并且笛卡尔积满足交换律。
这样约定之后，关系具有以下六个性质：

1. **列同质性**：每一列中的分量是同一类型的数据，来自同一个域
2. **列名唯一性**：不同的列可出自同一个域，但各列必须有不同的属性名
3. **列序无关性**：列的顺序可以任意交换
4. **行唯一性**：任意两个元组不能完全相同（关系是元组的集合，集合中元素不重复）
5. **行序无关性**：行的顺序可以任意交换
6. **分量原子性**：每一个分量必须是不可分的数据项

### 2.1.5 码（Key）

**候选码**（Candidate Key）：简称**码**，关系中能唯一标识一个元组的一个或一组属性，且不含多余属性（最小性）

比如说，学生关系（学号, 姓名, 性别, 专业号, 年龄）中，"学号"是一个候选码，因为每个学生的学号是唯一的，并且不含多余属性；而"姓名"不是候选码，因为可能有重名的人；"学号和姓名"虽然能唯一标识一个学生，但它包含了多余属性（姓名），因此也不是候选码。

再比如，选修关系（学号, 课程号, 成绩）中，"学号和课程号"是一个候选码，因为每个学生选修每门课程的成绩是唯一的，并且不含多余属性；而"学号"或"课程号"单独都不是候选码，因为一个学生可以选修多门课程，一个课程也可以被多个学生选修。

最极端的情况：由关系中的所有属性构成的属性组是这个关系的候选码，称为 **全码**（All-key）

**主码**（Primary Key）： 从候选码中选定的一个，用于唯一标识元组
主码是SQL里才有的概念，在创建基表时可以为基表定义‘主码’也可以不定义主码。在一张基表中，最多只能定义一个主码。

**主属性**（Prime Attribute）候选码中的属性（包含在任意候选码中的属性）
**非主属性**（Non-prime Attribute）不包含在任何候选码中的属性

### 2.1.6 关系模式（Relation Schema）

关系是元组的集合，关系模式是关系的型，元组集合是关系的值。

**关系模式**是对关系的描述，即关系的**结构定义**。
关系的关系模式一般是稳定不变的，但关系的值通常会随着时间的变化而变化。
关系中一个确定的元组集合被称为是该关系的一个**值**，也被称为是一个关系实例(relation instance)，它描述了关系在某个确定时刻的取值。

**完整形式**：R(U, D, dom, F)

- R：关系名
- U：属性名集合 {A₁, A₂, ..., Aₙ}
- D：属性所来自的域
- dom：属性与域的映像集合
- F：属性间数据的依赖关系集合

**简化形式**：R(U) 或 R(A₁, A₂, ..., Aₙ)

### 2.1.7 关系数据库（Relational Database）

- **关系数据库的型**：关系数据库模式（所有关系模式的集合），是对关系数据库的描述
- **关系数据库的值**：关系数据库（某一时刻所有关系的值的集合）

关系数据库中，**实体**及**实体间的联系**都用关系来表示。

## 2.3 关系的完整性

**关系的完整性**：对关系的某种约束条件，是关系的**值**及其随时间变化时应该满足的一些约束条件。关系在任何时刻都要满足这些约束要求。

关系模型中可以定义**三类完整性约束**：

| 类型                       | 说明               | 是否强制           |
| -------------------------- | ------------------ | ------------------ |
| **实体完整性**       | 主码不能为空       | 是（关系的不变性） |
| **参照完整性**       | 外码值有效         | 是（关系的不变性） |
| **用户定义的完整性** | 特定应用的语义约束 | 视需要而定         |

> 实体完整性和参照完整性是关系模型必须满足的完整性约束，被称为"**关系的两个不变性**"，由关系系统自动支持。

### 2.3.1 实体完整性（Entity Integrity）

**规则 2.1 实体完整性规则**：

> 实体完整性（Entity Integrity）是指关系中元组（二维表中的行）的唯一性。若属性 A 是基本关系（基表）的主码中的属性，则属性 A **不能取空值**（null）。

**空值**的含义：就是"不知道"或"不存在"或"无意义"的值。
![1775716189274](image/lec2关系数据库/1775716189274.png)
**实体完整性规则说明**：

1. 实体完整性规则是针对基本关系（表）而言的，一个基本表通常对应现实世界中的一个实体集
2. 现实世界中的实体是可区分的，即它们具有某种唯一性标识
3. 在关系数据库管理系统中，以"主码"作为基本表中元组（实体）的唯一性标识
4. 主码中的属性不能取空值——若取空值则说明存在某个不可标识的实体，与第 2 点矛盾

### 2.3.2 参照完整性（Referential Integrity）

#### 关系间的引用

在关系模型中，实体及实体间的联系都是用关系来描述的，因此自然存在着关系与关系间的引用。

**三类引用情形**：

| 例子                             | 引用描述                                                                                                               |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| 例 2.1（实体间的引用）           | 学生（**学号**, 姓名, 性别, `<u>`专业号`</u>`, 年龄）引用了专业（**专业号**, 专业名）的主码"专业号"    |
| 例 2.2（联系与相关实体间的引用） | 选修（**`<u>`学号`</u>`**, **`<u>`课程号`</u>`**, 成绩）引用了学生的主码"学号"和课程的主码"课程号" |
| 例 2.3（实体集内部的引用）       | 学生（**学号**, 姓名, 性别, 专业号, 年龄, `<u>`班长学号`</u>`）引用了自身的主码"学号"                        |

#### 外码（Foreign Key）

**定义 2.5 外码**：

> 设 F 是基本关系 R 的一个或一组属性，但不是关系 R 的码。如果 F 与基本关系 S 的主码 Kₛ 相对应，则称 F 是关系 R 的**外码**（Foreign Key）。
>
> - 基本关系 R 称为**参照关系**（Referencing Relation）
> - 基本关系 S 称为**被参照关系**（Referenced Relation）或**目标关系**（Target Relation）

**注意**：

- 关系 R 和 S 不一定是不同的关系（可以自引用）
- 目标关系 S 的主码 Kₛ 和参照关系的外码 F 必须定义在**同一个域上**
- 外码并不一定要与相对应的主码同名（但习惯上同名，以便识别）

**参照图示例**：

```
学生关系 --专业号--> 专业关系              （a）

学生关系 <--学号-- 选修关系 --课程号--> 课程关系   （b）

学生关系 <--班长学号-- 学生关系（自引用）          （c）
```

#### 参照完整性规则

**规则 2.2 参照完整性规则**：

> 若属性（或属性组）F 是基本关系 R 的**外码**，它与基本关系 S 的主码 Kₛ 相对应，则对于 R 中每个元组在 F 上的值必须为：
>
> - **或者取空值**（一个元组在 F 中每个属性上的取值均为空值）
> - **或者等于 S 中某个元组的主码值**

**例说明**：

- 例 2.1：学生关系中"专业号"属性（外码）只能取两类值：
  - 空值：表示尚未给该学生分配专业
  - 非空值：必须是专业关系中某个元组的"专业号"值（不可能被分配到**不存在的专业**）
- 例 2.3：学生关系中"班长学号"属性（外码）只能取两类值：
  - 空值：表示该学生所在班级尚未选出班长
  - 非空值：必须是本关系中某个元组的学号值

### 2.3.3 用户定义的完整性（User-defined Integrity）

**含义**：针对某一具体关系数据库的约束条件，反映某一具体应用所涉及的数据必须满足的语义要求。

关系模型应提供定义和检验这类完整性约束的机制，以便使用统一的系统方法处理它们，而不是由应用程序来承担这一功能。

**常见的用户定义完整性约束**：

- **唯一性约束**：某属性的值不能重复（如候选码的定义方式）
- **非空约束**：某属性的值不能为空（如非主属性'课程名'不能为空）
- **取值范围约束**：某属性只能取特定值域中的值（如'学分'属性只能取值 {1, 2, 3, 4}，'性别'只能取 {男, 女}）

**综合示例**：
学生（学号, 姓名, 性别, 身份证号, 院系代码, 专业代码, 录取大类代码）

| 约束类型       | 内容                                                                                                  |
| -------------- | ----------------------------------------------------------------------------------------------------- |
| 实体完整性     | 主码：学号（不能为空，且唯一）                                                                        |
| 参照完整性     | 外码：院系代码、专业代码、录取大类代码（引用对应关系的主码）                                          |
| 用户定义完整性 | 唯一性约束：身份证号；非空约束：姓名、性别、录取大类代码、身份证号；取值范围约束：性别只能取 {男, 女} |

## 2.4 关系代数

### 2.4.1 关系代数概述

是一种抽象的查询语言，它用**关系运算**来表达对关系的操作。
在关系代数中，运算对象是关系，运算结果亦为关系

关系代数的运算符有两类：传统的**集合运算符**和专门的**关系运算符**
传统的集合运算是从关系的“水平”方向，即行的角度进行
专门的关系运算不仅涉及行而且涉及列

**关系代数表达式**
以关系为运算对象，使用关系运算经**有限次复合**后形成的**表达式**称为 ‘关系代数表达式’
可以用关系代数表达式来表示关系模型上的**数据操纵**（包括数据查询、数据插入、删除和修改操作）

![1775801780753](image/lec2关系数据库/1775801780753.png)
![1775801858938](image/lec2关系数据库/1775801858938.png)

#### 关系代数中使用的记号

| 符号    | 含义                                 |
| ------- | ------------------------------------ |
| A, B, C | 属性名                               |
| a, b, c | 对应的属性值                         |
| X, Y, Z | 关系中的一个属性组                   |
| R, S, T | 关系名                               |
| R(U)    | 关系模式：关系名R，关系中的属性集合U |
| head(R) | 关系R的表头（属性名集合）            |
| body(R) | 关系R的表体（元组集合）              |
| r, s, t | 关系中的一个元组                     |
| t[A]    | 元组t在属性A上的分量值               |

设有一个n目关系R(A₁, A₂, ..., Aₙ)，其中A₁, A₂, ..., Aₙ是关系R的属性

- 它的一个关系实例也设为R
- 关系模式可以表示为$R(A_1, A_2, ..., A_n)$或$head(R)=\{A_1, A_2, ..., A_n\}$
- $t\in R$表示t是关系R中的一个元组
- $t[A_i]$表示t在属性Aᵢ上的分量值
- 若$A=\{A_{i_1}, A_{i_2}, ..., A_{i_k}\}$，其中$A_{i_1}, A_{i_2}, ..., A_{i_k}$是关系R的属性，那么A称为关系R的一个属性组/属性集，记为$A\subseteq head(R)$
- $t[A]$表示t在属性组A上的分量值集合，即$t[A]=\{t[A_{i_1}], t[A_{i_2}], ..., t[A_{i_k}]\}$
- $\overline{A}$表示从$\{A_1, A_2, ..., A_n\}$中去掉$\{A_{i_1}, A_{i_2}, ..., A_{i_k}\}$后剩余的属性组，即$\overline{A}=head(R)-A$

#### 元组的连接

R为n目关系，S为m目关系。

设$t_r\in R,\ t_s\in S$，则$\widehat{t_r t_s}$称为**元组的连接**。
$\widehat{t_r t_s}$是一个n+m目元组，前n个分量与$t_r$的分量相同，后m个分量与$t_s$的分量相同。
为方便表示，用$(t_r, t_s)$来表示$\widehat{t_r t_s}$。

#### 象集

给定一个关系R(X, Z)，其中X和Z是关系R的属性组
对属性组X定义域上的一个值$x_0$，他在R中的象集定义为：$Z_X_0=\{t[Z]|t\in R\ and\ t[X]=x_0\}$。

这表示R中属性组X取值为$x_0$的元组在属性组Z上的取值集合。
![1775803183991](image/lec2关系数据库/1775803183991.png)
![1775803273641](image/lec2关系数据库/1775803273641.png)
另一个属性组不说的时候默认取补集

#### 关系模式的命名规则

在同一个数据库（模式）中，关系名**互不相同**
在不同的数据库（模式）之间，允许存在同名、但互不相关的关系

在**同一个关系**中：**属性名互不相同**，不同的属性可以有相同的属性域
在不同的关系之间，一般遵循同名同义，异名异义的原则
如果有不符合的，可以用赋值运算来解决

#### 相容表

如果关系R和关系S的关系模式满足下述要求，则称关系R和S是**相容**的（或者称关系R和S是‘相容表’、‘同类关系’、相同的关系模式）

- 相同的目（属性的数目）
- R和S的属性一一对应，并且属性对应的域和语义都相同
  ![1775804349099](image/lec2关系数据库/1775804349099.png)

#### 赋值运算符

$R(A_1, A_2, ..., A_n) := <expression>$
这个expression是一个关系代数表达式，表达式的结果是一个关系，或者是一个关系。
这个赋值表达式可以理解为

- 为右边的数据操纵请求定义了一个**查询关系**，用以保存其操纵结果；
- $R$是该查询关系的关系名，$A_1, A_2, \ldots, A_n$是查询关系的属性名；
- 查询关系也可以被作为**访问对象**，参与后续关系代数表达式的表示。
  ![1775804775138](image/lec2关系数据库/1775804775138.png)

赋值运算可以用来保存中间变量，简化复杂的关系代数表达式的表示。
可以给关系重命名，还可以对关系的属性进行重命名
![1775804917707](image/lec2关系数据库/1775804917707.png)

### 2.4.2 关系代数的基本运算

#### 并（Union）

R和S有相同的目n，并且对应的属性具有相同的域
$R \cup S = \{t | t \in R \ \vee t \in S\}$

#### 差（Difference）

R和S有相同的目n，并且对应的属性具有相同的域
$R - S = \{t | t \in R \ \wedge t \notin S\}$
![1775805516993](image/lec2关系数据库/1775805516993.png)

#### 笛卡尔积（Cartesian Product）

设有两个关系：
R: n目关系，k1个元组
S: m目关系，k2个元组
笛卡尔积R × S
结果关系的关系模式：(n + m) 列
前n列来自于关系R，后m列来自于关系S
结果关系中的行：(n + m)元组，共有 k1 × k2个
每个元组由R中的一个元组和S中的一个元组连接而成
![1775805606817](image/lec2关系数据库/1775805606817.png)
任意两个关系，都可以进行笛卡尔积运算
笛卡儿积 满足 交换律 和 结合律

如果在关系R和S中存在同名的属性，在执行它们之间的笛卡尔积运算时，将被认为是两个**不同的属性**。
在笛卡尔积的结果关系中，对每一对同名属性，必须至少对其中的一个进行**换名**
![1775805742531](image/lec2关系数据库/1775805742531.png)

#### 选择（Selection）

选择，又称为限制
选择运算符：$\sigma_F(R)$ 即 R WHERE F
表示在关系R中选出满足条件F的元组组成一个新的关系
F是选择条件，是一个逻辑表达式，取值为真或假
F可以是：

- 基本形式：A θ a 或 A θ B，其中A和B是关系R的属性，a是一个常量，θ是一个比较运算符（=, <, >, <=, >=, <>）
- 符合形式：在基本的选择条件上可以进一步进行**逻辑运算**（由若干个择条件通过逻辑运算符（与、或、非）经有限次组合而成）

选择是在行的角度的运算
![1775806008715](image/lec2关系数据库/1775806008715.png)
选择的结果关系模式：与关系R相同
![1775806076956](image/lec2关系数据库/1775806076956.png)

#### 投影（Projection）

从R中选出**属性组A**，得到一个新的关系
$\pi_A(R)$ 即 SELECT A FROM R
从列的角度的运算
![1775806258026](image/lec2关系数据库/1775806258026.png)
如果结果关系中存在重复的元组，则只保留一个（关系是集合，集合中元素不重复）
![1775806501921](image/lec2关系数据库/1775806501921.png)

### 2.4.4 关系代数扩充运算

**关系完备系统**

- 采用关系数据结构，并且支持并、差、笛卡尔积、选择、投影等五
  个关系代数的基本运算符的系统，被称为是关系完备的系统。
- 一个关系完备的系统，能够支持所有的关系操作。

#### 关系代数的扩充运算

##### 交

对关系R和S，有相同的目n，并且对应的属性具有相同的域
$R \cap S = \{t | t \in R \ \wedge t \in S\}$
![1775889726007](image/lec2关系数据库/1775889726007.png)
![1775890355636](image/lec2关系数据库/1775890355636.png)
注意这里不能用下面那种表示，因为选择会选出一整个元组，这样得到的关系相交的结果必然是空集，因为元组不重复

##### 连接

连接又称为$\theta$连接
用于将关系 R 和关系 S 根据连接条件 F 连接起来，得到一个新的关系
$R \bowtie_F S = \{t | t = (r, s), r \in R, s \in S, F(r, s) \text{为真}\}$
F是连接条件，是一个逻辑表达式，取值为真或假
![1775890723996](image/lec2关系数据库/1775890723996.png)
等值连接：连接条件 F 是一个或多个属性之间的等值比较
自然连接：两个关系之间的所有同名属性对之间的等值连接，并且同名关系只保留一份在结果关系里
外连接：在结果集中同时含有自然连接结果元组和悬浮元组
![1775890990529](image/lec2关系数据库/1775890990529.png)
![1775891093754](image/lec2关系数据库/1775891093754.png)
这里B是同名属性，所以要换名

###### 等值连接

等值连接的连接条件是一个或多个属性之间的等值比较
![1775891216458](image/lec2关系数据库/1775891216458.png)

###### 自然连接

是一种特殊的等值连接，它要求两个关系之间的所有同名属性都必须相等，并且在结果关系中只保留一份同名属性。
一般的连接操作是从行的角度进行运算，自然连接还要消除重复列，所以它还涉及列的运算。
![1775891309154](image/lec2关系数据库/1775891309154.png)
![1775891333242](image/lec2关系数据库/1775891333242.png)

###### 关系的自连接

自连接不是运算符，是指一个关系和它自己进行连接运算的情况
在执行关系的自连接时，由于参与连接运算的是同一个关系R，它们具有相同的关系名和关系模式，无法直接使用常见的‘θ-连接’或‘自然连接’来表示。
此时，需要使用**赋值运算**来为原关系创建一个临时的中间关系。在该中间关系与原关系之间：

- 关系模式相同但关系名不同
- 对应属性的属性名可以相同、也可以不同
  ![1775906969156](image/lec2关系数据库/1775906969156.png)

###### 外连接

两个关系R和S进行自然连接时，如果R中的某个元组r在S中没有与之匹配的元组s，那么r就会被舍弃，这些被舍弃掉的就称为**悬浮元组**。
![1775910689108](image/lec2关系数据库/1775910689108.png)

如果把悬浮元组也保留下来，并且在结果关系中用空值（NULL）来填充那些没有匹配的属性值，那么得到的连接结果就称为**外连接**。
左外连接：保留左关系R中的所有元组
右外连接：保留右关系S中的所有元组
全外连接：保留R和S中的所有元组
![1775910787693](image/lec2关系数据库/1775910787693.png)
![1775910797181](image/lec2关系数据库/1775910797181.png)
![1775910827134](image/lec2关系数据库/1775910827134.png)

##### 除运算

设关系R除以关系S的结果为关系T，则T包含所有在R中但不在S中的属性及其值，且T的元组与S的元组的所有组合都在R中。
$T:=R \div S$，其中R是被除关系，S是除关系，T是商关系
“T包含所有在R中但不在S中的属性及其值”，那么head(T) = head(R) - head(S)
T的元组与S的元组的所有组合都在R中，说明$\forall t \in T, \forall s \in S, (t, s) \in R$
![1775913151258](image/lec2关系数据库/1775913151258.png)
![1775913186021](image/lec2关系数据库/1775913186021.png)
也就是说，$T = \{t | t \in \pi_{head(R) - head(S)}(R) \wedge \forall s \in S, (t, s) \in R\}$
